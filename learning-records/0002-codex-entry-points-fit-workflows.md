# Codex entry points fit existing workflows

The learner explained that beginners should usually start with Codex App because it exposes the full workflow: choose a project, open a thread, give a task, inspect the plan and execution, review the diff, and decide whether to accept the result. They also explained that CLI, IDE Extension, and Web/Cloud should be chosen based on the user's existing coding workflow and where the code lives.

**Evidence**: The learner mapped CLI to command-line users, IDE Extension to editor-centered coding, and Web/Cloud to GitHub-hosted code, then summarized that the best entry point is the one that most naturally fits the user's prior workflow.
