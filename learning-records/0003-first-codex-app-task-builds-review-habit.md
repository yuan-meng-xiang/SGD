# First Codex App task builds the review habit

The learner explained that a first Codex App demonstration should use a blank or safe practice project, open a fresh thread, and assign a small task with clear boundaries such as a simple Hello World webpage. They emphasized that the teaching goal is to show the whole loop - understanding the task, making a plan, changing files, summarizing results, and then reviewing the review pane and file diff before accepting the work.

**Evidence**: The learner warned that starting with a large project makes the code too broad for a human to review, weakens control over the codebase, and can let an agent's misunderstanding destabilize the whole project.
