# Knowledge Graphs

This directory stores stage snapshots of the learner's current understanding.

The graph is not a complete encyclopedia. It is a teaching map built from the mission, lessons, reference cards, and learning records. Each snapshot should help the learner explain the topic to someone else and choose what to do in a real task.

## Rhythm

- Start each topic with a thin framework of 5-7 main nodes.
- Keep each lesson small and add learning records only when the learner demonstrates understanding.
- Every 3-5 lessons, create or revise a graph snapshot.
- Promote a snapshot to a stable reference only after it has survived review and real use.

## What To Capture

- Nodes: concepts the learner can explain or is deliberately preparing to learn.
- Edges: relationships such as belongs to, depends on, contrasts with, applies when, commonly confused with, or step before.
- Evidence: lesson links, reference links, learning records, or source material.
- Status: learned, introduced, planned, or needs verification.
