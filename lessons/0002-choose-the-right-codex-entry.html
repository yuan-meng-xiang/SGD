Lesson 0002

Codex 的四个入口怎么选

这一课只练一个能力：你能不能给新手讲清楚 App、CLI、IDE Extension、Web/Cloud 分别适合什么场景。

主教材：`Codex橙皮书.pdf` 第 16-18 页。官方当前文档也把 Codex 分成 App、CLI、IDE Extension、Web 等入口；具体安装方式和套餐包含情况以官方页面为准。

本课速记卡：Codex 入口选择速查卡 打开速记卡

入口不是能力等级

给别人讲 Codex 入口时，最容易讲乱的一点是：把 App、CLI、IDE、Web 讲成“高级程度不同”。更好的讲法是，它们对应不同工作场景。

App 管任务，CLI 管终端，IDE 管编辑器上下文，Web/Cloud 管远程仓库任务。

这句话够短，适合作为你讲第二课时的开场。

四个入口怎么讲

入口 一句话 适合谁
Codex App 桌面任务指挥台 想要图形界面、刚开始练习、本地项目管理、多线程和 review 的人。
Codex CLI 终端里的工程搭档 熟悉终端、Git、构建、测试命令，想让 Codex 直接在项目目录里工作的开发者。
IDE Extension 编辑器里的上下文助手 已经在编辑器里写代码，想让 Codex 利用打开文件、选区、文件引用来协作的人。
Web/Cloud 云端任务执行入口 希望连接 GitHub 仓库、让 Codex 在云端后台跑任务、产出 PR 或远程处理问题的人。

为什么新手先从 App 开始

教材建议本地项目、网页练习和日常开发先从 Codex App 开始。原因不是 App “更高级”，而是它更容易让新手看到完整工作流：选择项目、开 thread、输入任务、看计划、看执行过程、看 diff、决定是否接受。

App 建立直觉 CLI 学命令流 IDE 贴近日常编辑 Web/Cloud 处理远程仓库

所以给别人讲时可以说：先别急着比较哪个入口最好，先看你要在哪个地方完成任务。

选择入口的三个问题

  • 代码在哪里：本地文件夹，还是 GitHub 仓库？
  • 你在哪里工作最顺：桌面 App、终端、编辑器，还是浏览器？
  • 任务怎么验收：看本地 diff、跑测试命令、编辑器联动，还是生成 PR？

检索练习

你要给一个完全新手演示“选择项目、开 thread、看 diff、review 修改”的完整流程，最适合先用哪个入口？

今天要能说出口

请用 3-5 句话回答：如果一个人问“我该从 Codex 哪个入口开始”，你会怎么帮他判断？

写给新手听的版本

要包含两个点：为什么新手通常先从 App 开始，以及什么时候应该切到 CLI、IDE 或 Web/Cloud。

参考答案：如果是新手，我建议先从 Codex App 开始，因为它最容易看到完整流程：选项目、开 thread、交代任务、看计划、看 diff、做 review。等你熟悉终端和 Git 后，可以用 CLI 处理真实项目里的命令流；如果你主要在编辑器里改代码，就用 IDE Extension；如果任务在 GitHub 仓库里、希望 Codex 云端后台处理或产出 PR，就考虑 Web/Cloud。
