Lesson 0003

跑完 Codex App 的第一次任务闭环

这一课只练一个能力：你能不能把 Codex App 第一次上手讲成一条完整、可检查、低风险的任务流程。

主教材：`Codex橙皮书.pdf` 第 21-30 页。第 19-20 页的安装步骤会随版本变化，本课先不背安装按钮位置，而是掌握第一次打开后的核心工作流。

第一次不要做大项目

新手第一次用 Codex App，目标不是立刻做一个完整产品，而是跑通一条小闭环：让 Codex 进入一个安全练习项目，完成一个边界很小的任务，然后你能看懂它改了什么。

选练习项目，开清楚 thread，给小任务，看 diff 后再决定是否接受。

这句话能防止新手一上来就把重要项目、复杂需求、不可验收的任务全部塞给 Codex。

六个概念要讲清

概念 解释 新手最该记住
项目目录 Codex 要进入哪个文件夹工作。 第一次选干净练习目录，不要选重要项目或整个磁盘。
项目列表 你交给 Codex 的不同代码文件夹或仓库。 它不是聊天记录列表，而是代码项目列表。
thread 同一个项目里的一个任务对话。 一个明确任务开一个 thread，不要把所有事塞进一个对话。
任务窗口 你给任务、看计划、看执行过程、继续追问的地方。 任务越清楚，Codex 越容易做好。
review pane 检查 Codex 实际改动的地方。 不要只看文字总结，要看文件和改动。
diff 代码新增和删除的对比。 先看 diff，再决定是否接受或继续修改。

第一次推荐流程

登录 选练习项目 开 thread 输入小任务 看计划 看执行 打开 review pane 检查 diff

你给别人演示时，不要把重点放在“按钮在哪里”，而要让对方看到 Codex 的工作节奏：理解任务、行动、展示改动、等待你验收。

第一个任务要小

教材推荐的第一个任务很小，但很聪明：只做一个简单 HTML/CSS 页面。它能让新手看到文件被创建或修改、页面效果变化、diff 出现，同时风险很低。

示例任务：请帮我做一个简单网页，黑色背景，中间显示大字 Hello, Codex，字体白色，页面整体水平和垂直居中，只使用 HTML 和 CSS。

这个任务的价值不在页面本身，而在让用户学会“交代任务 -> 查看过程 -> 检查改动 -> 决定下一步”。

检索练习

Codex App 第一次完成任务后，新手最应该先看什么，而不是只相信文字总结？

今天要能说出口

请用 4-6 句话回答：如果你给一个新手演示 Codex App 的第一次任务，你会怎么安排，为什么不能一上来就做大项目？如果你卡住，就回到Codex App 第一次任务速查卡，只看“第一次流程”和“第一个任务”。

写给新手听的版本

要包含三个点：先选安全练习项目；一个明确任务开一个 thread；任务完成后检查 review pane 和 diff。

参考答案：第一次演示 Codex App，我会先准备一个干净的练习项目，而不是直接打开重要工作项目。然后为一个明确的小任务新建 thread，例如只用 HTML 和 CSS 做一个 Hello, Codex 页面，让新手看到 Codex 如何理解任务、制定计划、修改文件和总结结果。任务完成后，我不会只看 Codex 的文字总结，而是打开 review pane 检查 diff，确认新增和删除是否符合预期。这样做的目的，是先建立完整闭环和验收习惯，再逐步进入真实项目。
