Reference 0002

Codex 入口选择速查卡

入口不是能力等级，而是工作场景。先问“我要在哪里把任务推进完”。

一句话

App 管任务，CLI 管终端，IDE 管编辑器上下文，Web/Cloud 管远程仓库任务。

四个入口

入口 最好这样讲 适合场景
Codex App 桌面任务指挥台 新手上手、本地项目、多线程、看 diff、管理任务
Codex CLI 终端里的工程搭档 熟悉终端、Git、脚本、构建测试、真实项目命令流
IDE Extension 编辑器里的上下文助手 正在 VS Code、Cursor、Windsurf、JetBrains 等编辑器里改具体文件
Web/Cloud 云端任务执行入口 连接 GitHub 仓库、后台并行任务、生成 PR 或远程处理仓库问题

新手路线

1. App 2. CLI 3. IDE Extension 4. Web/Cloud

先用 App 建立项目、thread、diff、review 的直觉；再用 CLI 学会真实项目命令流；之后根据个人工作方式补 IDE 或 Web/Cloud。

三个判断问题

  • 代码在哪里：本地文件夹，还是 GitHub 仓库？
  • 你在哪里工作最顺：桌面 App、终端、编辑器，还是浏览器？
  • 任务要怎么验收：看本地 diff、跑命令、编辑器联动，还是生成 PR？
