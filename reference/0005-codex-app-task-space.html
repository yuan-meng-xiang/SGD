Codex App 任务空间速查卡

Codex App 不是一个大聊天框，而是围绕项目和 thread 管理工程任务的工作台。

一句话

项目决定 Codex 能看哪里，thread 决定这次只做什么，归档让任务列表保持干净。

三栏布局

区域 作用 讲解重点
左侧 项目、任务列表、搜索、入口导航 用来找项目和管理不同 thread
中间 任务对话窗口 用来下指令、看计划、追问和引导
右侧 多功能区域 看文件、diff、浏览器、注释等上下文工具

项目对话 vs 非项目对话

使用项目

让 Codex 围绕某个代码文件夹或仓库工作。适合改代码、读项目、检查 diff、写 README。

不使用项目

作为普通问答或想法讨论。适合问概念、比较方案、准备提示词，不适合直接改文件。

thread 管理原则

  • 一个明确任务开一个 thread。
  • 后续发现新问题，另开 thread，而不是把所有事塞进旧 thread。
  • 任务太多时用搜索找关键词。
  • 完成或暂时不用的 thread 及时归档。
  • 等待批准时先看清请求内容，再决定是否允许。
