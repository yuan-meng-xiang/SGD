# Notes

- The learner wants to teach Codex to others, so each lesson should include a concise explanation frame and at least one retrieval exercise.
- The course should be based on `Codex橙皮书.pdf`, but the PDF is non-official and time-sensitive details need official/current verification.
- The learner prefers tool subjects to grow into a knowledge graph over time. Use a thin framework first, teach small concrete lessons, then redraw a stage snapshot every 3-5 lessons.
- Future learning records should still stay short, but when useful they should mention connections to existing concepts, changed understanding, or the gap they expose for the next lesson.
- Knowledge graph snapshots live in `knowledge-graphs/`; they are learning maps, not final encyclopedias.
- Socratic follow-up is optional. After the learner answers the question left at the end of a lesson, review the answer first, then ask whether they want to start Socratic follow-up for that lesson; do not start it automatically.
- When Socratic follow-up is enabled, ask 2-3 focused questions that test definition, contrast, evidence, transfer, boundary, graph connection, or teaching clarity. Then summarize the learner's revised understanding and only create a learning record if there is evidence of genuine understanding.
- After the learner reviews and approves a set of workspace changes, automatically commit the approved changes and push them to the configured remote repository.
